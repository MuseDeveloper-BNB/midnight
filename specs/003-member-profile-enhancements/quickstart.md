# Quickstart Guide: Enhanced Member Profile

**Feature**: Enhanced Member Profile  
**Date**: 2025-01-28  
**Status**: Complete

## Prerequisites

Same as the base application: Node.js 20.x LTS, PostgreSQL 14+, npm/yarn. See [001-news-website quickstart](../001-news-website/quickstart.md) for initial setup, `.env`, database, and dev server.

## Feature Setup

### 1. Database Migration

The `SavedItem` model and migration (`prisma/migrations/..._add_saved_item/`) are in place. With PostgreSQL running:

```bash
# From repo root
npx prisma migrate dev --name add_saved_item   # applies migration if pending
npx prisma generate
```

If the DB was unavailable during setup, run `npx prisma migrate dev` once the DB is up to apply the migration.

### 2. Permissions

Extend `src/lib/permissions.ts` so `MEMBER` includes:

- `edit:own:profile`
- `read:own:comments`
- `save:content`

Use these in profile and saved actions/services as per [api-profile](./contracts/api-profile.md) and [api-saved](./contracts/api-saved.md).

### 3. Seed Data (optional)

`prisma/seed.ts` includes `SavedItem` rows for `member@midnight.local` (news + blog). Run `npx prisma db seed` to populate. Sign in as member, open `/profile` Saved section and published article pages to test Save/Unsave.

### 4. Run Application

```bash
npm run dev
```

- **Profile**: `/profile` (route under `(member)` group). Sign in as MEMBER (e.g. `member@midnight.local`), then open profile to use edit form, My comments, and Saved.
- **Save/Unsave**: Open a published news or blog article; use the Save/Unsave control, then check `/profile` Saved section.

## Verification

1. **Profile edit**: Change name/email on profile, submit; see success and updated values.
2. **My comments**: Post comments on news/blog; open profile; see them in "My comments" with links to articles.
3. **Saved**: Save articles from detail pages; open profile; see them in "Saved". Unsave from profile or article; list updates.
4. **Empty states**: Profile with no comments shows "You haven't commented yet"; no saved items shows "No saved articles yet".
5. **Responsive**: Profile layout works on mobile and desktop; matches existing design tokens.
6. **Loading (FR-011)**: Profile fetch, form submit, save/unsave, and pagination show loading indicators; no unexplained blank or unresponsive UI.
7. **Accessibility (FR-012)**: Profile, form, lists, and Save button are keyboard-navigable and meet WCAG 2.1 Level AA (focus, labels, contrast).
8. **Rate limiting (FR-013)**: Excess profile update or save/unsave attempts receive a clear, non-blocking message.
9. **Error + retry (FR-014)**: Save/unsave failures show an explicit error and optional retry; no silent failure.
10. **Clean design (FR-015)**: No layout shift on load or pagination; consistent spacing and section hierarchy; no redundant primary actions.
